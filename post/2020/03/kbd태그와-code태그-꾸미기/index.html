      # MarkDown 글쓰기

      마크다운 글쓰기가 좋은 점은 빠르게 형식 있는 문서를 작성할 수가 있기 때문입니다. 특히 개발자들이나 coding 관련 정보를 나눌 때는 두 배로 좋습니다. 마크다운 문서는 마크업 문서를 함께 지원하기 때문에 복잡한 것은 HTML 태그 형태의 표현도 가능합니다.

      마크다운이 유명해 지기 시작한 것은 제 기억엔 위키문서 작성이 유행하면서 부터였던 것 같습니다. 빠르고 편리하게 정해진 형태의 문서를 작성할 수 있기 때문에 각종 레퍼런스 문서나 지식정보 위주의 문서위주로 빠르게 성장해 왔습니다.

      거기에 각종 차트나 UML 형태를 지원하는 에디터도 점점 많아지고 있는걸 보니, 마크다운이 확실히 점점 대세로 자리잡아 가는 것 같습니다.

      그래서인지 요즘은 마크다운 에디터를 활용 하는 사람도 점점 늘어나고 있고, 티스토리 같은 블로그에서도 마크다운을 지원하기 시작했습니다. 그리고 스태틱 웹이라 불리는 지킬이나 뷰프레스 같은 도구 들은 마크다운 위주로만 글을 쓰기도 합니다.

      마크다운에 관련한 이야기는 포스팅을 따로 해야겠네요. 이야기가 점점 산으로 흘러가고 있으니 정신 차리고 본래 하려던 이야기로 돌아가겠습니다.

      # 태그 활용을 더 예쁘게

      그렇게 삽입된 태그를 좀더 예쁘게 꾸며주면서 전달력을 높이기 위해 kbd 태그와 code 태그를 활용합니다. 하지만 그냥 입력해서는 전혀 느낌이 오지 않죠. 안이쁘니까요.

      그래서 예쁘게 꾸며줌과 동시에 가독성을 올리기 위해 css를 추가해 주기로 합니다.

      /* 코드 span CSS 효과 */
code {
  color: #900;
  background-color: #ddd;
  padding: 0 0.5em;
  font-family: Consolas, Monaco, 'Andale Mono', 'Ubuntu Mono', monospace;
  font-size: 0.9em;
  display: inline-block;
}

pre>code {
  line-height: 1.4 !important;
  tab-size: 4;
  border: 1px solid #ccc;
  border-radius: 5px;
    padding: 15px !important;
}

/* 키보드 효과 css */
kbd {
  background-color: #f7f7f7;
  border: 1px solid #ccc;
  border-radius: 3px;
  box-shadow: 0 1px 0 rgba(0, 0, 0, 0.2), 0 0 0 2px #fff inset;
  padding: 0.1em 0.5em;
  margin: 0 0.1em;
  text-shadow: 0 1px 0 #fff;
  white-space: nowrap;
  font-family: Consolas, Monaco, 'Andale Mono', 'Ubuntu Mono', monospace;
  font-size: 0.9em;
  display: inline-block;
}
